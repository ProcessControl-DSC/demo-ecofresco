PC MES Kiosk

Extensión del kiosco de asistencia para la captura e imputación de horas a proyectos y tareas

Odoo 19 · LGPL-3 · Process Control
Nota: este manual se entregará con capturas de pantalla reales una vez validado el módulo en el entorno de staging del cliente. Las secciones marcadas con "CAPTURA" indican el lugar donde se insertarán.

1. ¿Qué hace este módulo?

El módulo pc_mes_kiosk amplía el kiosco nativo de Asistencias de Odoo con una pantalla táctil de selección de proyecto y tarea. Al fichar entrada o salida, el operario puede indicar a qué proyecto y tarea se deben imputar las horas de esa jornada. Cuando se registra la salida, Odoo crea automáticamente una línea de parte de horas vinculada al registro de asistencia.

El módulo no modifica el flujo de fichaje existente: actúa como una capa adicional que se intercala en el kiosco público estándar mediante un parche de componente Owl.

2. Requisitos

Módulo Motivo
hr_attendance Kiosco y modelo de asistencias. Base obligatoria.
hr_timesheet Modelo account.analytic.line para los apuntes de parte de horas.
project Modelos project.project y project.task.

3. Instalación

  1. Copiar el directorio pc_mes_kiosk en la ruta de addons del servidor.
  2. Actualizar la lista de módulos: Ajustes > Aplicaciones > Actualizar lista de apps.
  3. Buscar "PC MES Kiosk" e instalar.
  4. Si el módulo ya estaba instalado y se añaden campos nuevos, ejecutar -u pc_mes_kiosk.

4. Configuración

Ir a Ajustes > Asistencias. El bloque "MES Kiosco — Imputación a proyecto" muestra tres opciones:

Opción Valores Descripción
Momento de imputación en el kiosco Al registrar entrada / Al registrar salida Determina en qué evento aparece la pantalla de selección.
Mostrar selector de proyecto Activado / Desactivado Si está activo, se muestra la lista de proyectos disponibles.
Mostrar selector de tarea Activado / Desactivado Si está activo, tras seleccionar un proyecto se muestran sus tareas abiertas.
CAPTURA Bloque de configuración "MES Kiosco — Imputación a proyecto" en Ajustes > Asistencias
Parámetros técnicos: la configuración se almacena como parámetros del sistema (ir.config_parameter): pc_mes_kiosk.mode, pc_mes_kiosk.show_project y pc_mes_kiosk.show_task. El endpoint público los lee sin autenticación para que el kiosco funcione en modo kiosk (sesión pública).

5. Flujo de uso

5.1 Fichaje con selección al registrar entrada

  1. El empleado abre el kiosco y se identifica con PIN o tarjeta.
  2. El sistema registra la entrada y, en lugar de pasar directamente a la pantalla de bienvenida, muestra la pantalla de selección de proyecto.
  3. El operario pulsa el proyecto correspondiente.
  4. Si el selector de tarea está activo, aparece la lista de tareas abiertas de ese proyecto. El operario selecciona una tarea o pulsa "Solo proyecto" para imputar únicamente al proyecto.
  5. Tras confirmar, el kiosco pasa a la pantalla de bienvenida habitual.
  6. Al fichar la salida, el sistema crea automáticamente la línea de parte de horas con las horas trabajadas.

5.2 Opción "Omitir"

En cualquier punto de la selección el operario puede pulsar "Omitir". En ese caso la asistencia se registra sin imputación de proyecto ni tarea, y no se creará ninguna línea de parte de horas al fichar la salida.

5.3 Creación automática del apunte de parte de horas

Cuando se escribe el campo check_out en un registro de asistencia (bien desde el kiosco, bien desde el back-office), el módulo comprueba si hay un proyecto o tarea asignados y si aún no existe una línea de parte de horas vinculada. Si se cumplen las condiciones, crea la línea de forma idempotente.

6. Modelos y campos

hr.attendance (ampliado)

Campo Tipo Descripción
pc_project_id Many2one → project.project Proyecto seleccionado en el kiosco para la imputación.
pc_task_id Many2one → project.task Tarea seleccionada en el kiosco para la imputación.
pc_timesheet_line_id Many2one → account.analytic.line (readonly) Línea de parte de horas generada automáticamente al cerrar la asistencia.

7. Endpoints HTTP públicos

Endpoint Método Descripción
POST /pc_mes_kiosk/targets JSON Devuelve la lista de proyectos y tareas disponibles, junto con los flags de configuración (mode, show_project, show_task).
POST /pc_mes_kiosk/set_target JSON Escribe pc_project_id y/o pc_task_id en el registro de asistencia abierto del empleado indicado.
Ambos endpoints usan auth="public" para ser accesibles desde el kiosco en sesión pública, que es el modo habitual de despliegue de hr_attendance. El acceso a los datos se realiza con sudo() acotado a las operaciones necesarias.

8. Limitaciones conocidas

  • Un registro de asistencia admite una única imputación (un proyecto y una tarea). No se permite división de horas entre varios proyectos.
  • Si el empleado no tiene usuario Odoo vinculado, el campo user_id del apunte de parte de horas quedará vacío.
  • Las tareas en etapas plegadas (cerradas o canceladas) no aparecen en el kiosco.
  • La línea de parte de horas se crea al fichar la salida. Si se elimina manualmente y se vuelve a escribir check_out, no se vuelve a crear.
El endpoint /pc_mes_kiosk/targets devuelve todos los proyectos y tareas activos sin paginación. En instalaciones con catálogos muy grandes puede generar un payload elevado. Se recomienda revisar el volumen de registros antes de activar el módulo en producción.
